# Automated Torque Validation System

This directory highlights an automated Poka-Yoke (mistake-proofing) system designed to guarantee the integrity of fastened joints in industrial assemblies.

## Overview

In critical manufacturing assemblies, improper torque applied to a fastener can lead to catastrophic product failure. Relying solely on an operator to read a gauge or listen for a mechanical "click" is prone to human error.

This system acts as a digital intermediary between the operator, the physical smart-tool (e.g., electronic torque wrenches), and the manufacturing execution system.

## Key Features & Architecture

### 1. Serial Communication Integration
The core of the system is a Python-based driver that interfaces with industrial smart-tools.
- **Protocol Parsing:** The application opens serial ports (RS232/USB) and listens for the continuous data stream generated by the torque tool.
- **Data Extraction:** It parses the raw byte stream to extract the exact peak torque applied and the angle of rotation.

### 2. Real-Time Limit Evaluation
As soon as the tool fires, the system evaluates the incoming data against a predefined set of parameters.
- **Tolerance Windows:** The application checks if the applied torque falls within the strict Upper Control Limit (UCL) and Lower Control Limit (LCL) specified for the active component.
- **Go / No-Go Logic:** The software immediately categorizes the fastening event as a "Pass" or "Fail". This logic prevents the unit from moving to the next station if a screw is under-torqued or over-torqued.

### 3. Integrated UI Feedback
Connected directly to the **Universal Workstation GUI Framework**, the system provides instant feedback to the operator.
- A large, highly visible indicator turns green for a good rundown and red for a bad rundown.
- If a rundown fails, the screen locks out further operations until an authorized supervisor swipes their badge to acknowledge and clear the defect.

### 4. Traceability Logging
Every single fastening event is permanently recorded.
- **Local SQLite Storage:** Logs the exact timestamp, tool serial number, target torque, actual applied torque, operator ID, and Pass/Fail result.
- **Digital Twin Sync:** The result is instantly broadcast to the network via the architecture outlined in our [Digital Twin Architecture](../digital-twin-architecture/) component.

## Technical Stack
- **Language:** Python
- **Libraries:** `pyserial` (for RS232 communication), `sqlite3` (for logging), `PyQt` (for GUI integration).
- **Concepts:** Industrial IoT (IIoT), Edge Processing, Poka-Yoke.
